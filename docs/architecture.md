# Architecture

[English](architecture.md) | [日本語](architecture.ja.md)

Brinsfield (2013) Track B is a socsim ABM on a Watts–Strogatz organisational network. Each employee carries a six-motive probability vector and chooses VOICE or SILENCE each step; the cross-sectional Brinsfield motive distribution emerges as the steady state of an EMA learning dynamic.

## World state (`world.rs`)

`SilenceWorld` implements socsim's `WorldState`:

- `employees: BTreeMap<AgentId, Employee>` — sorted keys give a deterministic activation order.
- `teams: Vec<Team>` — supervisor openness, knowledge stock, per-team climate.
- `network: SocialNetwork` — Watts–Strogatz by default (Erdős–Rényi / Barabási–Albert as sensitivity).
- `issue_salience`, `climate_of_silence`, `org_performance`, `retaliation_this_step`.

`Employee` holds `level / tenure / team / private_concern / expression / fear / psych_safety / ivt_strength / neuroticism / extraversion / perceived_silence / motive_vec / voice_threshold`.

`MotiveVec6` (`motives.rs`) is the six-dimensional simplex over `{ineffectual, relational, defensive, diffident, disengaged, deviant}` with `primary()` (argmax), `entropy()`, `kl_to()`, `normalize()`, and `ema_update()`.

## Mechanisms (`mechanisms.rs`) — 11 across the 6-phase loop

| # | Mechanism | Phase | Role |
|---|-----------|-------|------|
| 1 | `issue_salience` | Environment | update σ(t); optional shock |
| 2 | `retaliation_event` | Environment | mark retaliated agents (defensive driver) |
| 3 | `fear_appraisal` | Decision | threat appraisal of fear f_i |
| 4 | `voice_decision_rule_{6,4,3}dim` / `voice_decision` (LLM) | Decision | **mutually exclusive** voice + six-motive assignment |
| 5 | `silence_spiral` | Interaction | ρ_i ← neighbour silence ratio (diffident driver) |
| 6 | `prefalse_cascade` | Interaction | flip to VOICE if neighbour-VOICE > θ_i |
| 7 | `org_performance` | Reward | knowledge stock (ineffectual/disengaged drain) |
| 8 | `motive_dynamics` | PostStep | **EMA update of `motive_vec`** (η + social pull) |
| 9 | `psafety_update` | PostStep | ψ_i ← ψ_i + η·voiced − ν·retaliated |
| 10 | `climate_silence` | PostStep | aggregate C(t) and per-team climate |
| 11 | `motive_metrics` | PostStep | snapshot org-performance proxy |

Updates are **synchronous**: the decision mechanisms snapshot all employees at step start, then batch-write the new expressions / motives.

## Decision modes

`config.DecisionMode ∈ {Llm, Rule6dim, Rule4dim, Rule3dim}`. The driver (`simulation.rs`) wires exactly one decision mechanism. The rule modes share one sign-constrained six-motive softmax; `rule_4dim` / `rule_3dim` then collapse the six rows into the Knoll-4 / Van-Dyne-3 carriers (a competing-model KL ablation). The LLM mode parses a six-motive JSON distribution from the model.

## RNG streams (`derive_seed`)

- `&[0]` — world init (employee attributes + network generation).
- `&[1]` — engine (scheduler + Bernoulli draws in rule mode + retaliation).
- `&[3, agent_id, t]` — LLM `(agent, t)` seed.

The `rule_*` modes are bit-reproducible (integration test `rule6_is_bit_deterministic`). The LLM mode is pseudo-determinised by `temperature=0` + the `(agent, t)` seed + the prompt→response cache.
