# アーキテクチャ

[English](architecture.md) | [日本語](architecture.ja.md)

Brinsfield (2013) Track B は，Watts–Strogatz 組織ネットワーク上の socsim ABM である．各従業員は 6 動機確率ベクトルを持ち，毎 step VOICE か SILENCE を選ぶ．横断的な Brinsfield 動機分布は，EMA 学習動的の定常状態として創発する．

## World state（`world.rs`）

`SilenceWorld` は socsim の `WorldState` を実装する．

- `employees: BTreeMap<AgentId, Employee>` — sorted key が決定論的な活性化順序を与える．
- `teams: Vec<Team>` — supervisor openness，knowledge stock，team 別 climate．
- `network: SocialNetwork` — 既定 Watts–Strogatz（感度分析で Erdős–Rényi / Barabási–Albert）．
- `issue_salience`，`climate_of_silence`，`org_performance`，`retaliation_this_step`．

`Employee` は `level / tenure / team / private_concern / expression / fear / psych_safety / ivt_strength / neuroticism / extraversion / perceived_silence / motive_vec / voice_threshold` を持つ．

`MotiveVec6`（`motives.rs`）は `{ineffectual, relational, defensive, diffident, disengaged, deviant}` 上の 6 次元 simplex．`primary()`（argmax）・`entropy()`・`kl_to()`・`normalize()`・`ema_update()` を提供する．

## Mechanism（`mechanisms.rs`）— 6 phase ループ上の 11 個

| # | Mechanism | Phase | 役割 |
|---|-----------|-------|------|
| 1 | `issue_salience` | Environment | σ(t) 更新；任意ショック |
| 2 | `retaliation_event` | Environment | 報復被影響者を記録（defensive 駆動） |
| 3 | `fear_appraisal` | Decision | 恐怖 f_i の脅威評価 |
| 4 | `voice_decision_rule_{6,4,3}dim` / `voice_decision`（LLM） | Decision | **排他** voice + 6 動機割当 |
| 5 | `silence_spiral` | Interaction | ρ_i ← 近傍沈黙比（diffident 駆動） |
| 6 | `prefalse_cascade` | Interaction | 近傍 VOICE 比 > θ_i で VOICE へ転換 |
| 7 | `org_performance` | Reward | 知識ストック（ineffectual/disengaged が劣化） |
| 8 | `motive_dynamics` | PostStep | **`motive_vec` の EMA 更新**（η + 社会的牽引） |
| 9 | `psafety_update` | PostStep | ψ_i ← ψ_i + η·voiced − ν·retaliated |
| 10 | `climate_silence` | PostStep | C(t) と team 別 climate を集計 |
| 11 | `motive_metrics` | PostStep | org-performance proxy をスナップショット |

更新は **同期的**: 意思決定 mechanism は step 開始時に全従業員をスナップショットし，新しい expression / motive を一括書き戻す．

## 意思決定モード

`config.DecisionMode ∈ {Llm, Rule6dim, Rule4dim, Rule3dim}`．ドライバ（`simulation.rs`）は意思決定 mechanism を 1 つだけ配線する．rule モードは 1 つの符号制約付き 6 動機 softmax を共有し，`rule_4dim` / `rule_3dim` は 6 行を Knoll 4 / Van Dyne 3 のキャリアへ collapse する（competing-model KL ablation）．LLM モードはモデルから 6 動機 JSON 分布を parse する．

## RNG ストリーム（`derive_seed`）

- `&[0]` — world init（従業員属性 + ネットワーク生成）．
- `&[1]` — engine（scheduler + rule モードの Bernoulli + 報復）．
- `&[3, agent_id, t]` — LLM `(agent, t)` seed．

`rule_*` モードは bit 完全再現（integration test `rule6_is_bit_deterministic`）．LLM モードは `temperature=0` + `(agent, t)` seed + prompt→response cache で擬似決定論化する．
