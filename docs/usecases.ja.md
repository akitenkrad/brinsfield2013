# ユースケース

[English](usecases.md) | [日本語](usecases.ja.md)

## 1. ベースライン 6 動機 run（LLM 不要）

```bash
cargo run --release -- run --decision-mode rule_6dim --t-max 48 --runs 3 --seed 42
uv run brinsfield-tools visualize --results-dir results/latest
```

`motive_mix_stack.png` で 6 領域積上げの動機時系列を，`silence_kl_timeseries.png` で沈黙率と KL-to-reference の軌跡を確認する．

## 2. competing-model ablation（Brinsfield 6 vs Knoll 4 vs Van Dyne 3）

```bash
cargo run --release -- ablate --decision-modes rule_6dim,rule_4dim,rule_3dim --runs 20
```

コンソールにモード別の平均 `KL(mix‖reference)` が表示される．`rule_6dim < rule_4dim < rule_3dim`（6 次元が Brinsfield 分布に最も近い）を期待する．

## 3. 感度スイープ

```bash
cargo run --release -- sweep \
    --psafety-learn 0.05,0.10,0.20 \
    --p-retaliate 0.02,0.05,0.10 \
    --motive-init-defensive 0.05,0.10,0.15,0.20 \
    --runs 30
uv run brinsfield-tools visualize-sweep --results-dir results/latest
```

`sweep_defensive_heatmap.png` は定常 defensive 比率の ψ 学習・報復への応答を，`sweep_motive_response.png` は 12.65% アンカー周りの引込み域を示す．

## 4. LLM 駆動 run（Ollama）

```bash
export OLLAMA_HOST=http://localhost:11434
export OLLAMA_MODEL=llama3.1
cargo run --release -- run --decision-mode llm \
    --llm-cache-path runs/brinsfield_cache.json --temperature 0.0 --t-max 48 --runs 5
```

同一引数で再実行すると warm cache から再生する（cache-hit ≈ 100%，`llm_meta.json` に記録）．

## 5. 実証側 6 因子再現（合成データ）

```bash
uv run brinsfield-tools survey-loader --synthesize-n 300 --sample synth
uv run brinsfield-tools reproduce --sample synth --synthesize-n 300
```

6 因子 CFA が 1–5 因子競合に優位であること，Study 4 の ΔR² 目標が満たされることを，実調査データなしで確認する．
